# 001 — Project Setup

## Purpose

This was the initial setup of the WorkspaceCanvas repository. It established the baseline project structure by creating a React frontend and a Django backend side by side in the same repository.

## What Was Added

- React + TypeScript + Vite frontend scaffolded in `frontend/`
- Tailwind CSS configured in the frontend
- Django project created in `backend/`
- Django `config/` folder created as the project configuration package (not a feature app)
- Root `.gitignore` configured for Python, Node, and environment files
- `README.md` created with project description and tech stack

## Files Involved

| File | Purpose |
|---|---|
| `frontend/package.json` | Frontend dependencies and scripts |
| `frontend/vite.config.ts` | Vite build configuration |
| `frontend/src/` | React application source |
| `backend/manage.py` | Django management entry point |
| `backend/config/settings.py` | Django settings |
| `backend/config/urls.py` | Root URL configuration |
| `backend/requirements.txt` | Python dependencies |
| `README.md` | Project overview |
| `.gitignore` | Files excluded from version control |

## How It Works

The project is a monorepo with two separate applications:

- `frontend/` — a React single-page application built with Vite
- `backend/` — a Django REST API

The `backend/config/` folder is the Django project configuration package generated by `django-admin startproject`. It contains settings, URL routing, and WSGI/ASGI entry points. It is not a feature app — it only holds project-wide configuration.

Feature apps (such as `accounts`) live directly inside `backend/` alongside `config/`.

## How To Run / Test

**Frontend:**

```bash
cd frontend
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173).

**Backend:**

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py runserver
```

Open [http://localhost:8000](http://localhost:8000).

## Important Decisions

**Monorepo structure:** Both frontend and backend live in one repository for simplicity at this stage. They are kept in separate folders so they can be split into separate repositories later if needed.

**`config/` as the Django project folder:** Django generates a project folder with the same name as the project. We named it `config/` rather than `workspacecanvas/` to make it clear that it holds configuration, not business logic.
